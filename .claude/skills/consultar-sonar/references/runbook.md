# SonarQube Local Runbook

## Recreate the container from scratch

Used when `sonarqube-local` no longer exists or its data is corrupted. This wipes any existing Sonar history.

```sh
docker rm -f sonarqube-local 2>/dev/null || true
docker volume rm \
  gol_app_mobile_sonarqube_data \
  gol_app_mobile_sonarqube_extensions \
  gol_app_mobile_sonarqube_logs 2>/dev/null || true

docker run -d --name sonarqube-local \
  -p 9000:9000 \
  -v gol_app_mobile_sonarqube_data:/opt/sonarqube/data \
  -v gol_app_mobile_sonarqube_extensions:/opt/sonarqube/extensions \
  -v gol_app_mobile_sonarqube_logs:/opt/sonarqube/logs \
  sonarqube:community
```

First login is `admin` / `admin`; the UI forces a password change.

## Generate a token

`http://localhost:9000/account/security` → *Generate Token* (type *Global Analysis Token*).

Persist it for the skill by either:
- exporting `SONAR_TOKEN=...` in the shell, or
- adding `# <token>` as a trailing comment line in `sonar-project.properties` at the repo root.

## Reset admin password (preserving data)

If the password is forgotten but the volume must stay:

```sh
docker exec -it sonarqube-local bash -lc 'cd /opt/sonarqube && java -cp "$(ls lib/jdbc/h2/*.jar)" org.h2.tools.RunScript -url "jdbc:h2:./data/sonar" -user sonar -password sonar -script /dev/stdin' <<'SQL'
UPDATE users SET crypted_password='$2a$12$uCkkXmhW5ThVK8mpBvnXOOJRLd64LJeHTeCkSuB3lfaR2N0AYBaSi', hash_method='BCRYPT', reset_password=true, user_local=true WHERE login='admin';
SQL
```

The hash above corresponds to the plaintext `admin`. Restart the container afterwards.

## Common ports / paths

- API base: `http://localhost:9000/api`
- Web API explorer: `http://localhost:9000/web_api`
- Container name: `sonarqube-local`
- Volumes: `gol_app_mobile_sonarqube_{data,extensions,logs}`
- Repo config: `sonar-project.properties` (at repo root)
- Coverage feed: `coverage/lcov.info` (generated by `yarn test:coverage`)
